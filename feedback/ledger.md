# 피드백 원장 (Claude가 누적 관리)

**후기 소스 = 채팅 (유일 경로)**
사용자가 이 채팅에 직접 평가를 적는다. 사이트의 후기란은 본인 기록용일
뿐이며 Claude에게 전달되지 않는다(정적 사이트 한계).

Claude는 채팅으로 후기를 받는 즉시 아래에 시간순으로 누적하고,
**매 회차 1단계(주제 선정) 전에 이 파일 전체를 읽어** 그동안 쌓인
모든 피드백을 누적 반영한다.

형식:
```
## [YYYY-MM-DD] 프로젝트 N · 별점 X/5
원문: <사용자가 쓴 그대로>
해석/패턴: <Claude가 뽑아낸 선호·회피 신호>
다음에 반영: <구체적 조치>
```

## [2026-05-16] 프로젝트 1 ORBIT · 별점 3/5
원문: 제대로 작동하고 마우스 원클릭이라는 아이디어는 훌륭했으나 난이도가
2단계 3단계 상승될때 플레이어의 컨트롤의 영향을 덜 받는 방식으로 난이도가
상향되어서 플레이어의 어떤 컨트롤적인 역량보다도 난이도 상승되는 방식의
운에 의해 점수가 결정되는 경우가 많음. (플레이를 계속 해도 결국 운빨이라는
느낌을 받게 하여 플레이하고자 하는 욕망이 생기지 않음).

해석/패턴:
- 좋아함: 단순한 원클릭/직관 조작, 정상 동작.
- 싫어함(핵심): 난이도 상승이 "회피 불가능한 무작위 위협 증가"로 구현되어
  플레이어 에이전시(통제력)를 깎음 → 결과가 운으로 결정 → 숙련 보상 없음
  → 반복 동기 상실.
- 선호 원칙(앞으로 모든 회차에 적용):
  1. 점수는 **플레이어의 컨트롤 실력**이 지배 변수여야 한다.
  2. 난이도 상승 = 무작위 위협 증가가 아니라 **정밀도·타이밍·속도 요구치
     상승**. 위협은 항상 예고되고(telegraphed) 완벽한 플레이로 회피 가능
     해야 한다(공정성/도달가능성 보장).
  3. 결정론적 — 같은 입력이면 같은 결과. 운으로 죽지 않는다.
  4. 숙련 곡선: 계속하면 분명히 더 잘하게 되는 체감.

다음에 반영: 2회차는 "운 불가, 실력 전부"가 핵심인 게임으로. 모든 위협을
예고하고, 다음 통과 지점이 현재 속도에서 반드시 도달 가능하도록 스폰을
제약(도달가능성 불변식). 난이도는 속도↑·간격↓로 정밀도 요구만 올린다.
→ [반영 완료] 2회차 WEAVE로 구현·검증(도달가능성 불변식 테스트 통과).
   이 선호 원칙(운 배제·숙련 보상)은 이후 모든 회차에 계속 적용한다.

## [2026-05-16] 프로젝트 2 WEAVE · 별점 3/5
원문: 점점 빨라지고 통과간격은 좁아지며 난이도가 점차 상승되는 점은
좋았고, 그게 다라서 아쉽다. RPG 스테이지 중간보스 보스 스테이지가
있듯이 기믹이 있었으면 좋았을 거 같다. 그리고 각각 프로젝트를 만들 때마다
토큰이 어떻게 소모되어 있는지 볼 수 있게 해줘.

해석/패턴:
- 좋아함: 점진적 난이도 곡선 자체는 만족(운 배제 원칙은 유효).
- 부족함: 단조로움 — 한 가지 메커니즘 반복. **구조와 변주**를 원함:
  스테이지 진행감, 중간보스/보스 같은 특수 국면(기믹), 클리어 보상.
- 누적 선호 원칙(앞으로 계속):
  · (1회차) 운 배제·숙련 보상 — 유지.
  · (신규) 단순 반복 금지. 스테이지/페이즈 구조 + 예고된 특수 기믹으로
    리듬에 굴곡을 준다. 단, 기믹도 운 아닌 실력으로 돌파 가능해야 함.
- 운영 요구: 프로젝트마다 토큰 소모 내역을 가시화(갤러리에 표시).

다음에 반영:
1) WEAVE에 스테이지 구조 + 중간보스/보스 기믹 추가(모두 예고·도달가능,
   운 0% 유지). 클리어 보상.
2) 갤러리 각 카드에 "제작 토큰(추정)" 내역 표시 + CLAUDE.md 기록.
→ [반영 완료] WEAVE 개정: 8문/스테이지, 3배수 MINI-BOSS, 6배수 BOSS
   (HP바·+10 보너스), 예고된 이동 빈틈(추적가능 cap). 검증 29/29.
   갤러리 카드에 "🔧 제작 토큰(추정)" 추가, CLAUDE.md에 회차별 기록.
   누적 원칙(운 배제 + 단조로움 금지·구조/기믹)은 이후 회차에 계속 적용.
